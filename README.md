sql-sales-performance-analysis
Sales Performance Analysis (SQL)
 Project Overview

This project analyzes sales transaction data to understand revenue trends, customer behavior, and business performance using SQL.

The analysis focuses on identifying top-performing categories, regions, customers, and monthly revenue patterns.

 Dataset Description

The dataset contains sales order information including:

Order details (order ID, date, quantity)

Financial metrics (amount, profit)

Product hierarchy (category, sub-category)

Customer and location details (state, city)

Payment methods

 Tools Used

PostgreSQL

SQL (aggregation, filtering, CASE statements)

pgAdmin

 Key Business Questions Answered

Which states generate the highest revenue?

Which product categories are most profitable?

What are the monthly revenue trends?

Who are the top 5 customers by total spend?

How many orders are profitable vs loss-making?

Which payment methods are most commonly used?

 Key Insights

Revenue is concentrated in a few top-performing states.

Certain product categories contribute disproportionately to total revenue.

Monthly revenue shows clear fluctuations, indicating seasonal demand.

A small number of customers contribute significantly to total sales.

Majority of orders fall under profitable transactions.

 Sample SQL Techniques Used

GROUP BY and aggregation functions

CASE statements for classification

Date extraction and formatting

Sorting and filtering for insights
